# 🔥 AI 简历拷打机 V2

上传简历，找出**最容易被面试官问穿的项目描述**，检查量化指标与 GitHub 证据，并生成中文 PDF 报告。

> 🔒 **作者看不到你的简历。** 项目只在你的电脑上运行，没有作者服务器、账号、数据库或遥测。使用 Ollama 本地模型时，简历内容不会离开本机。

## Windows：三步启动

1. 点击 GitHub 右上角 **Code → Download ZIP**。
2. **完整解压 ZIP**，不要在压缩包预览里直接运行。
3. 双击 **`启动中文版.bat`**。

中文文件名显示异常时，双击 **`START_WINDOWS.bat`**，功能完全一样。

第一次启动会自动创建本地环境并安装依赖。窗口不会闪退；发生错误时会停留并生成 `启动日志.txt`。

## 先体验 Demo

打开网页后点击：

> **⚡ 立即体验 Demo（无需上传）**

系统会使用一份完全虚构的简历，展示风险评分、证据缺口、技术追问和 PDF 报告。

## 它会检查什么？

- “准确率提升 37.8%”是否缺少 baseline、数据划分和原始日志
- “独立完成”“主导开发”是否没有明确个人贡献边界
- 论文、奖项和竞赛排名是否缺少官方验证入口
- 简历声明与 GitHub 代码、测试、benchmark 和结果文件是否一致
- 哪些描述存在**疑似夸大或高追问风险**

**高风险不等于造假。** 工具只能提示证据缺口，不能替代背景调查或人工判断。

## 本地模型（可选）

- 推荐：`qwen2.5:7b-instruct`
- 低配置：`llama3.2:3b`
- 不安装模型也能直接使用规则引擎

## 系统要求

- Windows 10/11、macOS 或 Linux
- Python 3.10+
- 首次安装依赖时需要网络

## 常见问题

### 双击后仍然无法启动

打开项目目录中的 `启动日志.txt`。也可以右键空白处打开终端，运行：

```powershell
.\START_WINDOWS.bat
```

### 浏览器没有自动打开

启动窗口会显示类似地址：

```text
http://127.0.0.1:xxxxx
```

复制到浏览器即可。

### 如何停止？

关闭启动窗口，或在窗口中按 `Ctrl + C`。
